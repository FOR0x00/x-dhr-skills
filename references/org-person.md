# 智能人事模块

## 当前范围

检索人员信息：检索我的个人信息；检索他人信息，根据用户输入识别用户要检索的目的，转化为查询条件，执行查询

## 查询我的个人信息

#### 1. 获取花名册表单Schema
调用 `get-org-person-form` 工具， 获取花名册表单 `Schema`

#### 2. 获取我的个人信息
调用 `query-org-person-me` 工具，获取当前用户个人信息，结合花名册表单 `Schema`，返回给用户。

## 检索人员信息

### 执行流程

#### 1. 获取花名册表单Schema

调用 `get-org-person-form` 工具， 获取花名册表单 `Schema`

#### 2. 识别用户查询意图，组装查询条件

严格遵守字段定义，正确匹配字段值的类型，和字段支持的查询操作符 `queryExpression`。

查询条件必须至少包含一个明确、可映射到表单字段、且能收敛结果范围的条件，例如姓名、手机号、工号、部门、岗位等。

遇到以下任一情况时，不要调用 `query-org-person`，必须先使用 `UserAsk` 追问更具体的查询条件：

- 没有任何查询条件。
- 查询条件只有“所有员工”“花名册”“人员信息”等宽泛描述。
- 查询条件无法唯一映射到 `Schema` 字段或字段不支持对应 `queryExpression`。
- 查询条件过宽，可能返回大量人员或泄露非必要花名册信息。

#### 3. 发起查询

查询条件合法且足够收敛后，向用户展示最终查询条件；并执行 `query-org-person` 工具完成检索
查询结果可能为空，花名册中没有匹配到，也可能当前用户无权查看。

### 错误边界

- 表单 schema 发生变化：以最新 schema 重新校验，保留仍可唯一映射的用户值，其余重新询问。
